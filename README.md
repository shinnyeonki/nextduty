# NextDuty - 근무 이동 알림 앱

이 프로젝트는 정해진 편성표에 따라 근무지를 이동해야 하는 사용자를 위한 자동 알람 서비스입니다.

## 파일별 책임 범위 (Responsibility)

### 1. `MainActivity.kt`
- **앱의 진입점**: 안드로이드 시스템과의 통신 및 초기 설정을 담당합니다.
- **권한 관리**: 알림 권한 등 필요한 시스템 권한을 요청합니다.
- **UI 컨테이너**: `DutyApp` 컴포저블을 실행하고 전역적인 앱 상태를 관리합니다.

### 2. `DutyCore.kt` (근무 로직 응집)
- **데이터 저장소**: 모든 근무 시간표(JU1, JU2) 데이터를 보유합니다.
- **비즈니스 로직**: 현재 시간과 설정값에 기반하여 현재 근무지, 다음 근무지, 퇴근까지 남은 시간 등을 계산하는 핵심 로직을 포함합니다.
- **순수 기능**: UI나 시스템 종속성 없이 오직 '데이터'와 '계산'에만 집중합니다.

### 3. `AlarmCenter.kt` (알람 시스템 응집)
- **알람 스케줄링**: `AlarmManager`를 사용하여 정확한 시간에 알람을 예약하거나 취소합니다.
- **알림 관리**: 알람 발생 시 상단 알림창(Notification)을 띄우고 관리합니다.
- **알람 동작**: 알람 소리 설정, 진동, 풀스크린 인텐트 실행 등 알람의 모든 동작 방식을 정의합니다.

### 4. `DutyScreens.kt`
- **사용자 인터페이스**: 앱의 모든 화면 UI(설정 화면, 모니터링 화면)를 Compose로 정의합니다.
- **상태 표시**: `DutyCore`에서 계산된 정보를 시각적으로 표현합니다.

### 5. `AlarmActivity.kt`
- **전용 알람 화면**: 알람이 울릴 때 잠금 화면 위에서도 즉시 나타나는 전체 화면 UI입니다.
- **사용자 확인**: 사용자가 근무 이동을 인지하고 '확인' 버튼을 누를 수 있는 인터페이스를 제공합니다.

### 6. `AlarmReceiver.kt`
- **이벤트 수신기**: 시스템이 예약된 알람 시간에 도달했을 때 신호를 받아 `AlarmCenter`를 호출합니다.

### 7. `PreferenceManager.kt`
- **설정 저장소**: 사용자가 선택한 근무 조, 편성표 번호, 서비스 활성 상태 등을 `DataStore`를 통해 영구 저장합니다.
