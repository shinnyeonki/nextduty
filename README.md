# NextDuty - 근무 이동 알림 앱

이 프로젝트는 정해진 편성표에 따라 근무지를 이동해야 하는 사용자를 위한 자동 알람 서비스입니다. 사용자가 어떤 상황(화면 꺼짐, 다른 앱 사용 중 등)에서도 5분 전 이동 알림을 명확히 인지하도록 설계되었습니다.

## 파일별 책임 범위 (Responsibility)

### 1. `MainActivity.kt` (진입점 및 권한 관제)
- **권한 관리 센터**: 앱 작동에 필수적인 권한들(`알림`, `정확한 알람`, `전체 화면 알림`, `다른 앱 위에 표시`, `방해 금지 무시`)의 상태를 총괄하고 획득을 유도합니다.
- **UI 컨테이너**: `DutyApp` 컴포저블을 실행하고 `PreferenceManager`를 통해 저장된 앱 설정 상태를 UI와 동기화합니다.

### 2. `DutyCore.kt` (도메인 규칙 엔진 - 핵심 로직)
- **비즈니스 로직의 심장**: 근무 편성표, PT 예외 규칙, 출퇴근 시간 계산 등 모든 '업무 도메인 지식'을 중앙 집중 관리합니다.
- **이동 시점 스케줄링**: 단순히 근무 시작만이 아니라, **'장소가 변경되는 모든 시점'**(점심시간 이동, 종료 5분 전 퇴근 준비 등)을 계산하여 알람 데이터를 생성합니다.

### 3. `AlarmReceiver.kt` (이벤트 브로드캐스트 및 강제 실행)
- **트리거 중계**: 시스템 알람 시간에 신호를 받아 `AlarmCenter`(알림 생성)를 호출합니다.
- **방해 모드 실행**: 화면이 켜져 있을 때도 사용자를 확실히 깨우기 위해 '다른 앱 위에 표시' 권한을 활용하여 `AlarmActivity`를 즉시 강제로 화면에 띄웁니다.

### 4. `AlarmCenter.kt` (시스템 인프라 조절기)
- **스케줄링 실행**: `AlarmManager`를 사용하여 `DutyCore`가 계산한 시간에 시스템 알람을 예약/취소합니다.
- **시스템 알림(Notification)**: `Full-Screen Intent`를 포함한 고우선순위 알림을 생성하여 화면 꺼짐 상태에서도 알람이 작동하도록 합니다.

### 5. `AlarmActivity.kt` (전용 알람 인터페이스)
- **통합 알람 화면**: 잠금 화면 위나 다른 앱 사용 중에 나타나 사용자에게 이동 정보를 시각적으로 알리고 알람 소리를 재생합니다.

### 6. `BootReceiver.kt` (시스템 복구)
- **알람 복원**: 기기 재부팅 시 `PreferenceManager`에 저장된 설정을 바탕으로 알람 스케줄을 자동으로 재등록합니다.

### 7. `PreferenceManager.kt` (데이터 퍼시스턴스)
- **영구 저장소**: 사용자의 근무 설정 및 앱 상태를 `DataStore`를 통해 안전하게 보관합니다.

### 8. `DutyScreens.kt` (UI 컴포넌트)
- **UI 선언**: 설정 화면 및 모니터링 화면 등 앱의 모든 시각적 요소를 Compose로 정의합니다.
